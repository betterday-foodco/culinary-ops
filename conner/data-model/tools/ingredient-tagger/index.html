ℹ️ How this works + workflow

Data source: ingredients-review.csv loaded on page open. Rows come from a join of the Neon Ingredient table (302 rows from production) with an extraction of allergen data from the SPRWT admin UI PDF export (144 rows).

Workflow:

  1. Rows marked matched have SPRWT suggestions — review and click Accept ✓ on each, or click Accept visible ✓ at the top to bulk-accept a filtered subset.
  2. Rows marked needs review have no SPRWT data — click the allergen chips yourself if you know what's in them.
  3. Rows marked sprwt only are ingredients that exist in the SPRWT PDF but NOT in the Neon Ingredient table. These can't be updated (no Neon id) — they're flagged for Darlene to migrate separately.
  4. Your edits are auto-saved to localStorage on every click. Refresh the page and your work is still there. Use Clear draft to start over.
  5. When done, click Download CSV to get the full edited dataset, or Download SQL to get UPDATE statements for just the changed rows. The SQL is safe to run against any Neon branch — it only touches the rows you explicitly edited.

Allergen list: the 13 canonical allergens from the SystemTag table (Peanuts, Tree Nuts, Eggs, Fish, Shellfish, Crustaceans, Soy, Sesame, Wheat, Mustard, Sulphites, Dairy, Gluten). Corn, Hemp, Chickpea, Lentil, Sunflower and other SPRWT-legacy "allergens" are intentionally excluded — those are being retired.

Dislikes: preserved in the CSV for future use but not written to Neon yet — the Ingredient table has no dislikes column. When Gurleen adds one, the captured values can be applied with a one-liner.

Running this tool: python3 -m http.server 8000 from anywhere (this folder or the repo root), then open http://localhost:8000/index.html if you're serving this folder directly, or navigate to the folder if you're serving from the repo root.
